Это веб-приложение я написал для контроля своего бюджета и расходов. Идея была в том, что бы избавиться от google-sheets в пользу решения с простым вводом данных и с простым представлением данных.

Приложение писалось под требования конкретного заказчика (им был я сам), но в процессе разработки я решил добавить простую авторизацию, для тестирования этого приложения любым желающим.

По факту использования приложения появляются моменты из области UX, которые заносятся в файл todo.txt и по мере возможности дорабатываются. В целом опыт использования и отзывы заказчика — положительные. :)

Приложение реализовано для мобильных устройств, как SPA, на чистом Java Script. Для работы с базой данных написал небольшие php скрипты под каждый тип запроса.

Цель проекта — получить практические навыки и закрепить теорию, в рамках изучения фрэймворка ASP.NET Core.

Функционал — регистрация, авторизация и CRUD — операции с данными, которые представленны в объекте Контакт. Данные хранятся и предоставляются API сервисом посредством HTTP запросов. Пара — логин пароль валидны как на сайте, так и в десктоп клиенте.

На сайте есть администратор — он может управлять созданием и удалением Юзеров и Администраторов. Юзер может видеть детали всех контактов, но редактировать и удалять только свои. Не зарегистрированный гость может только видеть список имён и аватарок контактов. Данные регистрации на сайте можно использовать для авторизации в десктоп клиенте, и наоборот.

Приложение концептуально задумано и реализовано для использования с мобильного телефона. Проект написан на React и Redux, использован шаблон create-react-app;

Color Code — это игра, в которую я играл в детстве. Пластмассовое игровое поле было разделено на зоны, где с помощью маленьких цветных фишек один игрок загадывал код, а второй пытался его угадать.

Проблема была у игрока, который загадывал код — ему было просто скучно, т.к. его ролью было придумать комбинацию и ждать варианты ответов второго игрока.

Я написал Color Code, т.к. очень любил эту игру. Сейчас код загадывается программой и игрок не зависит ни от кого.

Итак, на старте есть скрытый от нас код, являющийся комбинацией цветов. Наша задача, используя логику, за определённое число попыток разгадать эту последовательность.

После каждого хода мы получаем определённую информацию о том, насколько наш вариант соответствует «Секретному Коду».

Таким образом, делая ход и получая информацию мы решаем задачу и получаем «Секретный код».
